# ИИ-админ для Minecraft (Paper) сервера — инструкция по запуску

## Что у тебя получится
Python-скрипт следит за чатом сервера в реальном времени, отправляет
сообщения локальной нейросети (Ollama, бесплатно, без ключей и интернета),
и в зависимости от тяжести нарушения — пишет предупреждение, мьютит,
кикает или банит игрока через RCON.

Эскалация по умолчанию: предупреждение → публичное предупреждение →
мьют на 10 минут (при повторных нарушениях) → кик (травля/угрозы) →
бан (только для самых тяжёлых случаев — реклама вредоносных ссылок,
разжигание ненависти, угрозы насилием).

⚠️ **Важно понимать риски:** это автоматическая модерация без участия
человека. Локальная модель (`qwen2.5:3b`) гораздо слабее облачных ИИ и
может ошибаться — забанить за неудачную шутку или пропустить реальное
нарушение. Перед тем как доверять ей баны без проверки, рекомендуется:
- Несколько дней погонять только с `warn_chat`/`warn_private` (без
  mute/kick/ban), посмотреть в консоли, какие решения она принимает.
- Держать логи (`[РЕШЕНИЕ] ...` в консоли) под рукой, чтобы можно было
  разбанить игрока, если ИИ ошибся.
- Если сервер не только твой — предупредить игроков, что модерацию
  частично ведёт ИИ, и дать способ оспорить решение (например, написать
  тебе в Discord).

## Шаг 1. Поставь Paper вместо Vanilla

1. Скачай Paper здесь: https://papermc.io/downloads/paper
   (выбери версию, совпадающую с твоей текущей — например 1.21.x)
2. Останови текущий сервер.
3. Положи скачанный `paper-X.X.X.jar` в папку сервера, переименуй
   в то же имя, что было у твоего vanilla `server.jar` (или просто
   поменяй имя файла в своём `.bat`/`.sh` стартовом скрипте на новый jar).
4. Запусти сервер как обычно. Мир, прогресс, постройки — всё сохранится,
   Paper полностью совместим с vanilla-мирами.

## Шаг 2. Включи RCON

Открой `server.properties` в папке сервера, найди и поменяй:

```
enable-rcon=true
rcon.port=25575
rcon.password=ТВОЙ_ПАРОЛЬ_СЮДА
```

Поставь свой пароль вместо `ТВОЙ_ПАРОЛЬ_СЮДА` — любой, главное запомнить.
Перезапусти сервер, чтобы настройки применились.

## Шаг 2.5. Проверь EssentialsX (нужен для команды /mute)

Команда `kick` и `ban` есть в чистом Paper из коробки. А вот `/mute` —
команда плагина EssentialsX. Если у тебя он уже стоит (ты говорил, что
да) — всё ок, ничего делать не нужно. Если вдруг окажется, что нет,
скачай тут: https://essentialsx.net/downloads.html и положи .jar в
папку `plugins/` сервера, затем перезапусти его.

## Шаг 3. Установи Ollama (локальная нейросеть, бесплатно)

1. Скачай тут: https://ollama.com/download (Windows/Mac/Linux)
2. Установи как обычную программу.
3. Открой терминал/cmd и скачай лёгкую модель:

```
ollama pull qwen2.5:3b
```

Эта модель весит ~2 ГБ и спокойно работает на RTX 3060 8GB даже
параллельно с игрой и сервером — она не требовательная.

Проверь, что всё работает:
```
ollama run qwen2.5:3b "Привет, как дела?"
```
Если ответила — всё ок, можно выходить (Ctrl+D или /bye).

## Шаг 4. Установи Python-зависимости

```
pip install requests mcrcon pyyaml
```

## Шаг 5. Настрой и запусти скрипт

Скопируй файл `config.example.yaml` и переименуй копию в `config.yaml`
(в той же папке). Открой `config.yaml` и заполни:

- `log_path` — путь к файлу `logs/latest.log` внутри папки твоего сервера.
  Например: `C:\\Users\\Имя\\Desktop\\mc-server\\logs\\latest.log`
- `rcon_password` — тот же пароль, что ты вписал в `server.properties`.

`config.yaml` уже добавлен в `.gitignore` — он никогда не попадёт на
GitHub, так что пароль останется только у тебя на ПК.

Сохрани файл, затем запусти:

```
python ai_admin.py
```

Если всё настроено правильно, увидишь:
```
=== ИИ-админ запущен ===
Слежу за логом: ...
Жду события...
```

Зайди на сервер, напиши что-нибудь в чат — в консоли скрипта появится
лог обработки, а если "нарушение" — придёт сообщение в игре.

## Как это тестировать

Лучше тестировать постепенно, от мягкого к жёсткому:

1. Напиши обычное сообщение ("привет", "кто хочет копать алмазы") —
   должно быть полностью проигнорировано.
2. Напиши лёгкую грубость — должно прийти личное предупреждение (`tell`).
3. Напиши явный мат/оскорбление — публичное предупреждение в чат (`say`).
4. Повтори нарушение несколько раз тем же ником — должен сработать мьют
   (`mute`), и ты на 10 минут не сможешь писать в чат.
5. Кик/бан тестировать осторожно — после кика тебя выкинет с сервера,
   после бана придётся разбанивать себя через консоль сервера командой
   `pardon <твой_ник>` (это пишется прямо в окне консоли сервера, не
   через скрипт).

Следи за консолью скрипта — там видно `[РЕШЕНИЕ] ...` с полным
обоснованием (severity, action, счётчик нарушений) для каждого случая.

## Публикация на GitHub (через браузер, без терминала)

1. Зайди на https://github.com и нажми **New repository** (зелёная
   кнопка справа сверху или "+" в углу → New repository).
2. Дай имя, например `mc-ai-admin`, выбери **Public**, не отмечай
   "Add README" (он уже есть у нас) — жми **Create repository**.
3. На странице нового (пустого) репозитория нажми ссылку
   **"uploading an existing file"**.
4. Перетащи туда (или выбери через проводник) ВСЕ файлы:
   - `ai_admin.py`
   - `README.md`
   - `config.example.yaml`
   - `.gitignore`

   **Важно: НЕ загружай `config.yaml`** (если ты уже создал его у себя
   на ПК) — в нём твой реальный пароль RCON. На GitHub должен идти
   только `config.example.yaml`.
5. Внизу страницы напиши короткое описание коммита (например
   "Первая версия ИИ-админа") и нажми **Commit changes**.

Готово — репозиторий опубликован, и твой пароль в безопасности.

## Дальше можно расширять

- **Грифинг/действия игроков**: Paper позволяет ставить плагины,
  которые логируют ломание/постройку блоков (например CoreProtect).
  Дальше парсим уже не только чат, но и эти события — можно скинуть мне
  лог-формат CoreProtect и я допишу парсер.
- **История диалога**: сейчас оценивается текущее сообщение + счётчик
  прошлых нарушений, но без самого текста истории. Можно добавить
  буфер последних N сообщений игрока для более точного контекста.
- **Настройка порогов эскалации**: количество предупреждений перед
  мьютом, длительность мьюта (`MUTE_DURATION`) и формулировки правил
  (`SERVER_RULES`) — всё редактируется прямо в `ai_admin.py`.

## Если что-то не работает

- **Скрипт не видит события** → проверь, что путь к `latest.log` верный
  и файл реально обновляется при действиях в игре.
- **Ошибка RCON-соединения** → проверь `enable-rcon=true`, порт и пароль
  совпадают с `server.properties`, и сервер был перезапущен после правки.
- **Ollama долго отвечает** → попробуй модель `llama3.2:1b` — она ещё
  легче и быстрее, хоть и чуть менее умная.
